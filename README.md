# MVP: web-app на Google Apps Script для графика приёма лекарств

## Коротко об архитектуре
- **Frontend (HTMLService):** одна страница `Index.html` с крупными блоками: Профиль, Назначения, Быстрый ввод, Недельный график.
- **Backend (Apps Script):** API-функции `api*` в `Api.gs` + сервисный слой `AppService` + генератор расписания `SchedulerService`.
- **Хранение:** Google Sheets как БД, доступ через `SheetRepository` (DAL).
- **Расширение:** новые схемы добавляются в `SchedulerService.isDayIncluded`, новые экраны — как отдельные HTML-partials.

## Структура файлов
- `appsscript.json` — манифест проекта.
- `Config.gs` — карта листов и глобальные настройки.
- `Utils.gs` — проверки, обработка ошибок, id, даты.
- `SheetRepository.gs` — data access layer для Google Sheets.
- `Services.gs` — бизнес-сценарии (профиль, назначения, быстрый ввод, чтение расписания).
- `Scheduler.gs` — алгоритм генерации недельного плана.
- `Api.gs` — точки входа UI + API.
- `Index.html` — основной экран приложения.
- `Styles.html` — доступный UI (крупный шрифт, контраст, простые формы).
- `Client.html` — frontend-логика и вызов `google.script.run`.

## Принцип генерации расписания
1. Выбрать активные назначения пользователя.
2. Для каждого дня недели проверить активность назначения по датам.
3. Применить тип схемы:
   - `daily` — каждый день;
   - `interval` — каждые `everyDays`;
   - `weekday` — по списку дней недели `weekdays`.
4. Собрать строку расписания с дозировкой и маркерами контроля.
5. При наличии `companionRuleJson.enabled` добавить сопутствующий приём.
6. Сохранить результат в `ScheduleGenerated`.

## Базовые API-функции
- `apiInit()` — загрузка стартовых данных для экрана.
- `apiSaveUser(payload)` — создать/обновить профиль.
- `apiSavePrescription(payload)` — создать/обновить назначение.
- `apiQuickIntake(payload)` — быстро отметить факт приёма.
- `apiGetWeekSchedule(userId, weekStartIso)` — пересчитать и вернуть недельный график.

## UX для пожилых пользователей
- Крупный шрифт (20px+) и большие поля ввода.
- Высокий контраст и заметные кнопки действий.
- Минимум шагов: один экран, линейный сценарий сверху вниз.
- Ясные подписи и мгновенная обратная связь (toast).
